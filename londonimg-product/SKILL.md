# 倫敦印象 ｜ 商品頁文案產生器

幫同事為單一商品快速產出「可直接貼進 SHOPLINE 後台」的全套文案。

> ⚡ **速度鐵則（最優先）**：預設只產「文字版」（A／B／C）。這時**絕對不要開啟或讀取 `reference/block-html-templates.md`**——它是大檔，載入會明顯拖慢。**只有使用者明確說要「可貼 HTML／可貼語法」時**，才在產 HTML 那一步去讀它。其餘 reference 都是小檔、需要時再讀。

## 你會產出什麼（標準輸出）

分成兩大塊，**預設兩塊都給**：

> 產出順序：**A. 商品資訊 → B. 商品描述 → C. 商品頁設定**，分別對應教學頁的 **STEP 3 商品資訊 → STEP 4 商品頁語法 → STEP 5 商品頁設定**（教學頁選單前面還有 STEP 1 Claude Skill、STEP 2 圖片取得方式，是說明用，不對應產出）。

### A. 商品資訊（對應教學頁「商品資訊」分頁／後台「商品資訊」）
1. **商品名稱**（顯示在商品頁的名稱，也是頁面大標）— **一定要產兩個：商品名稱（繁體中文）＋商品名稱（英文）**（對應教學頁「商品資訊」分頁的兩欄，缺一不可）。去掉內部代號／測試前綴。
2. **商品摘要 = 頁面簡短介紹**（價格上方、使用者看得到那塊）。產出**一段 inline 樣式的促銷框 HTML**（樣式內嵌、不依賴外部 class，貼進商品摘要欄才會正常顯示）。第一行＝商品專屬一句賣點＋優惠（你產），第二行＝固定的湊免運促銷（照貼）：
   ```
   <div style="font-size:15px;line-height:1.85;color:#1c1b19">
     <div>〔商品一句賣點＋優惠，例：🎁 四角氣墊防摔 × 360°旋轉支架，晶透不泛黃〕</div>
     <div>🚚 湊免運嗎？<b><a href="https://www.londonimg.tw/pages/add" target="_blank">加價購</a>省更多｜超值好物一起帶！</b></div>
   </div>
   ```
   - 第一行**短、以一句賣點/優惠為主**，使用者掃一眼就懂；**詳細賣點、規格、情境一律不放這裡**，全部放下面的「商品描述」。
   - ⚠️ **第二行湊免運是全店固定，務必逐字原樣保留、絕不可省略或改寫**（含 🚚 emoji、`加價購` 連結 `https://www.londonimg.tw/pages/add`、`省更多｜超值好物一起帶！` 整句）。就算這個商品沒有特別促銷，這第二行也一定要在。輸出的商品摘要 HTML **永遠是「第一行＋這行湊免運」兩行**，缺一不可。

### B. 商品描述（對應教學頁「商品頁語法」分頁的 13 塊積木）
3. 依 `reference/page-blocks.md` 的**積木名稱逐塊產出文字**（與教學頁同名同順序），讓同事能一塊一塊對著填。
   - 每塊用 `【積木：◯◯】` 當小標，底下各欄位分別給程式碼框；**有版型之分的積木要標建議版型**（Hero A/B、棋盤格 A/B…判準見 page-blocks.md）。⚠️ **情境 橫圖壓文的 A/B 取決於「圖片暗部位置」、你沒看過圖**——不要亂猜建議版型，改輸出自選規則給同事：「圖暗部在左上→版本 A、暗部在下方→版本 B」。
   - **預設產這些**：Hero（直圖壓文）、重點 icon 列、特色區（多圖）、**情境 橫圖壓文**、**棋盤格**、收尾 CTA、規格、QA。
   - **只有真的用不到才略過**：影音（沒影片）、版本比較（沒有「一般款 vs 升級款」之類差異）。<b>不要因為「想不到」就略過情境 橫圖壓文/棋盤格</b>——幾乎所有商品都能寫。
   - **多圖橫移／SNS 社群圖牆＝預設不產**（skill 無從判斷有沒有多角度實拍圖、要不要放 IG 牆）：在結尾提醒一句「有多角度實拍圖想做圖牆、或想放 IG 牆（@london_apple），跟我說一聲我再補這兩塊」；使用者說要才產。
   - **加價購**＝全店固定共用，不產文字，只提醒同事照教學頁複製。
   - ⚠️ **舊版「定義句／特色區①②③④／對照表／版本卡片／使用情境／功能特色／適用機型表」已廢除，不要再輸出這些名稱**：定義句的 GEO 任務改由**收尾 CTA 的「一句描述」**承接；適用機型與相容性改在 **QA** 講（對應表見 page-blocks.md）。
   - 結尾提醒同事：到「商品頁語法」頁找**同名積木**，把文字填進 ✏️【…】；圖片自己上傳。**並提醒：若需要可直接貼進後台的 HTML，再跟我說「也給我可貼的 HTML」，我會分積木給。**
4.（選）**可直接貼 SHOPLINE 的填空 HTML** — 只有使用者要求時才產出。**務必逐積木分段給**（像教學頁那樣每塊一個獨立程式碼框、標出積木名稱與版型、可單獨貼），**不要給一整包**，讓同事按需挑用要哪些積木。
   - ⚠️ **每塊 HTML 一律照 `reference/block-html-templates.md` 對應積木（對應版型）的模板複製**（模板＝教學頁複製語法的原樣鏡像）：只把文字類 `✏️【…】` 換成你產的文字；**圖片網址、圖片說明（alt）、YouTube 嵌入網址、貼文／商品連結等佔位照模板原樣保留**（各積木佔位字不同，別改寫；alt 由同事換圖時自填）。**兩個例外要直接代入**：① SNS 的 `✏️【@你的IG帳號】`＝`@london_apple`、`✏️【你的 IG 網址】`＝`https://www.instagram.com/london_apple/`；② 收尾 CTA 兩顆按鈕的網址佔位——使用者已給商品網址就代入「完整網址＋#sl-product-image」（兩處同值），未知才保留佔位並提醒補填。
   - **絕對不可自創版面、不可改任何 style、不可加 font-family**。增減卡片／列／圖照模板內註解複製或刪一整段。
   - 細節另見 `reference/shopline-output.md` §「若要產出可貼 HTML」。

### C. 商品頁設定（對應教學頁「商品頁設定」分頁／後台「網路搜尋最佳化」四欄）
5. **標題**（meta title）
6. **簡介**（meta description）— ⚠️ 這是給 **Google 搜尋結果**用的，**不會顯示在商品頁上**、使用者瀏覽看不到。所以寫約 150 字、SEO 導向即可，不必擔心「太長沒人看」。
7. **關鍵字**（keywords，半形逗號分隔）
8. **描述性URL**（英文 slug）＋ 301 轉址提醒

## 流程

1. **先確認商品資訊**。若使用者沒給齊，主動問：
   - 商品名稱／型號（例：YU 360°無段支架手機殼 B67；任何品類都可：保護貼、Watch 錶帶、AirPods 殼、充電、掛繩、支架、旅行配件、清潔工具…）
   - 所屬自有品牌或代理品牌（YU / Londonimg / J/arch. / LaPO / 其他）
   - 3–5 個賣點／功能
   - 適用機型／相容裝置（**有相容性的品類才需要**，例：iPhone 12–16 mini 除外、Watch 42/46mm、AirPods Pro 2；掛繩、清潔工具等通用品答「通用／不挑機型」即可）
   - 材質、規格（可選）
   - 目標關鍵字（可選；沒給就由你依品類推斷）
   - 有無促銷（滿額免運、任選折扣等，可選）
   - 是否也要產出「可貼 SHOPLINE 的 HTML」（預設否）

   ⚠️ **開始寫商品描述前，先確認下面三題都有答案**。使用者訊息裡**已明確回答的題目免再問**（產出前一行複述確認即可）；**還沒答案的題目一定要停下來問**，不要自行假設：
   1. **有沒有商品影片（YouTube）？幾支？** → 有才產「影音」積木（英文小標＋區塊標題＋影片標題）；沒有就略過該積木。**未問到就先停下來問，不要直接假設沒有。**
   2. **適用機型／相容裝置有哪些？**（通用品答「不挑機型」即可）→ 不確定就標「請確認」，不要亂編。有相容性的品類，答案會寫進收尾 CTA 的一句描述與 QA；**通用品則不要硬寫機型**，CTA 一句描述改寫品類＋適用對象。
   3. **有沒有「版本／款式差異」可比較？**（例：一般款 vs 升級款、3.0 vs 一般、雙向吸 vs 單向）→ 有才產「版本比較」積木（① 對比合成圖的文字＋② 選哪款卡）；沒有就略過、不要硬湊。⚠️ 版本比較不做 HTML 表格。

   三題都有答案後，才開始產出。

2. **讀規則檔**，嚴格遵守（⚡ 為了快，**只讀當下需要的**，不要一次全讀）：
   - **平常產文字版（A／B／C）只讀這 4 個小檔**：
     - `reference/page-blocks.md` — 商品描述的區塊與欄位（B 部分照這個產）
     - `reference/brand-voice.md` — 品牌語氣與定位
     - `reference/seo-geo-rules.md` — SEO/GEO 各欄位寫法與字數（C 部分）
     - `reference/shopline-output.md` — SHOPLINE 平台限制與輸出格式
   - ⚡ **`reference/block-html-templates.md`（檔案大）只有在使用者明確要「可貼 HTML」時才讀**；不需要可貼 HTML 就**完全不要載入**它（這是加快產出的關鍵）。

3. **依「標準輸出」順序產出**，每個欄位都用獨立程式碼框（方便一鍵複製）。

4. **產出後自我檢查**（見各規則檔的檢查清單），尤其：
   - 標題沒塞促銷字、沒花式字體
   - 簡介重點前置、約 150 字
   - slug 全英數小寫、無中文/空格/特殊符號（如 °）
   - 文案語氣符合品牌（專業選物、不靠折扣轟炸）

## 重要原則

- **通用於全店所有品類**（官網實際分類）：手機殼／玻璃貼・鏡頭貼／MagSafe 磁吸周邊／行動電源／PD 快充線・快充頭／AirPods 保護殼／Apple Watch 錶帶・錶框・保護貼／iPad／手機掛繩＆夾片／手機・平板支架／車用配件…都用同一套 13 積木流程（各品類的適用範圍寫法見 seo-geo-rules.md「品類快查表」）。各規則檔的範例多以 B67 手機殼示意——**範例是「長相示範」不是品類限制**，照商品品類替換內容即可；**「適用機型」相關要求只適用於有相容性的品類**，通用品一律改寫品類＋適用對象、不要硬編機型。
- **繁體中文**輸出。
- **用同事看得懂的白話**：不要丟 H2／CSS／HTML／meta／slug 等術語（後台欄位就用它的中文名：標題／簡介／關鍵字／描述性URL）。要 HTML 時直接給「可貼的成品」，不用解釋語法。
- **據實**：不確定的規格不要編造，請使用者確認。
- 促銷放「**頁面簡短介紹**」或商品描述的湊單區，**不要進標題**、也不必硬塞進 SEO 簡介。
- **分工清楚**：頁面簡短介紹要短（優惠為主）；詳細內容（賣點/規格/情境/FAQ）全放商品描述。SEO 簡介是給 Google、不顯示在頁面。
- 若要產 HTML，務必遵守 `shopline-output.md`（用 px、描述欄勿放 `<style>`、影片用 iframe、FAQ 平鋪不折疊）。
