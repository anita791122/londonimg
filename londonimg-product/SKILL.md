# 倫敦印象 ｜ 商品頁文案產生器

幫同事為單一商品快速產出「可直接貼進 SHOPLINE 後台」的全套文案。

## 你會產出什麼（標準輸出）

分成兩大塊，**預設兩塊都給**：

> 產出順序＝教學頁選單順序：**A. 商品資訊 → B. 商品描述 → C. 商品頁設定**。

### A. 商品資訊（對應教學頁「商品資訊」分頁／後台「商品資訊」）
1. **商品名稱**（顯示在商品頁的名稱，也是頁面大標）— 可直接用「標題」，去掉內部代號/測試前綴。
2. **商品摘要 = 頁面簡短介紹**（價格上方、使用者看得到那塊）。產出**一段 inline 樣式的促銷框 HTML**（樣式內嵌、不依賴外部 class，貼進商品摘要欄才會正常顯示）。第一行＝商品專屬一句賣點＋優惠（你產），第二行＝固定的湊免運促銷（照貼）：
   ```
   <div style="font-size:15px;line-height:1.85;color:#1c1b19">
     <div>〔商品一句賣點＋優惠，例：🎁 四角氣墊防摔 × 360°旋轉支架，晶透不泛黃〕</div>
     <div>🚚 湊免運嗎？<b><a href="https://www.londonimg.tw/pages/add" target="_blank">加價購</a>省更多｜超值好物一起帶！</b></div>
   </div>
   ```
   - 第一行**短、以一句賣點/優惠為主**，使用者掃一眼就懂；**詳細賣點、規格、情境一律不放這裡**，全部放下面的「商品描述」。
   - 第二行湊免運是**全店固定**，原樣保留。

### B. 商品描述（對應教學頁「商品頁語法」分頁的各區塊）
3. 依 `reference/page-blocks.md` 的**區塊名稱逐塊產出文字**（首屏 Hero／定義句／特色區／使用情境／常見 QA…），讓同事能一塊一塊對著填。
   - 每塊用 `【區塊：◯◯】` 當小標，底下各欄位分別給程式碼框。
   - **預設產這些**：首屏 Hero、定義句、特色區①-④、**使用情境**、**功能特色**、適用機型、規格、常見 QA。
   - **只有真的用不到才略過**：影音實測（沒影片）、對照表／版本卡片（沒有「一般款 vs 升級款」之類差異）。<b>不要因為「想不到」就略過使用情境/功能特色</b>——幾乎所有商品都能寫。
   - 結尾提醒同事：到「商品頁語法」頁找**同名區塊**，把文字填進 ✏️【…】；圖片自己上傳。
4.（選）**可直接貼 SHOPLINE 的填空 HTML** — 只有使用者要求時才產出

### C. 商品頁設定（對應教學頁「商品頁設定」分頁／後台「網路搜尋最佳化」四欄）
5. **標題**（meta title）
6. **簡介**（meta description）— ⚠️ 這是給 **Google 搜尋結果**用的，**不會顯示在商品頁上**、使用者瀏覽看不到。所以寫約 150 字、SEO 導向即可，不必擔心「太長沒人看」。
7. **關鍵字**（keywords，半形逗號分隔）
8. **描述性URL**（英文 slug）＋ 301 轉址提醒

## 流程

1. **先確認商品資訊**。若使用者沒給齊，主動問：
   - 商品名稱／型號（例：YU 360°無段支架手機殼 B67）
   - 所屬自有品牌或代理品牌（YU / Londonimg / J/arch. / LaPO / 其他）
   - 3–5 個賣點／功能
   - 適用機型（例：iPhone 12–16，mini 除外）
   - 材質、規格（可選）
   - 目標關鍵字（可選；沒給就由你依品類推斷）
   - 有無促銷（滿額免運、任選折扣等，可選）
   - 是否也要產出「可貼 SHOPLINE 的 HTML」（預設否）

   ⚠️ **開始寫商品描述前，一定要先問這兩題**（就算上面資訊都齊也要問，因為無法從商品資訊推斷）：
   1. **有沒有商品影片（YouTube）？幾支？** → 有才產「影音實測」區塊（影片標題＋說明）；沒有就略過該區塊。**未問到就先停下來問，不要直接假設沒有。**
   2. **適用機型有哪些？** → 不確定就標「請確認」，不要亂編。

   問完這兩題、得到回覆後，才開始產出。

2. **讀規則檔**，嚴格遵守：
   - `reference/seo-geo-rules.md` — 各欄位的 SEO/GEO 寫法與字數
   - `reference/brand-voice.md` — 倫敦印象品牌語氣與定位
   - `reference/page-blocks.md` — **商品描述要對應的「商品頁語法」區塊與欄位**（C 部分照這個產）
   - `reference/shopline-output.md` — SHOPLINE 平台限制與輸出格式

3. **依「標準輸出」順序產出**，每個欄位都用獨立程式碼框（方便一鍵複製）。

4. **產出後自我檢查**（見各規則檔的檢查清單），尤其：
   - 標題沒塞促銷字、沒花式字體
   - 簡介重點前置、約 150 字
   - slug 全英數小寫、無中文/空格/特殊符號（如 °）
   - 文案語氣符合品牌（專業選物、不靠折扣轟炸）

## 重要原則

- **繁體中文**輸出。
- **用同事看得懂的白話**：不要丟 H2／CSS／HTML／meta／slug 等術語（後台欄位就用它的中文名：標題／簡介／關鍵字／描述性URL）。要 HTML 時直接給「可貼的成品」，不用解釋語法。
- **據實**：不確定的規格不要編造，請使用者確認。
- 促銷放「**頁面簡短介紹**」或商品描述的湊單區，**不要進標題**、也不必硬塞進 SEO 簡介。
- **分工清楚**：頁面簡短介紹要短（優惠為主）；詳細內容（賣點/規格/情境/FAQ）全放商品描述。SEO 簡介是給 Google、不顯示在頁面。
- 若要產 HTML，務必遵守 `shopline-output.md`（用 px、描述欄勿放 `<style>`、影片用 iframe、FAQ 平鋪不折疊）。
