# 商品頁語法 — 積木對照表（13 塊・與教學頁同名同順序）

> 產出商品描述時，**一律用下面的「積木名稱」分組**，並只給該商品用得到的積木。
> 同事拿到後，到教學頁「商品頁語法」找到同名積木，把文字填進對應的 ✏️【…】即可。
> 順序＝教學頁左側選單順序＝商品頁實際顯示順序。標「選用」的，沒有就略過、不要硬湊。
> ⚠️ **舊版區塊已全面改制**：「定義句」「對照表（HTML 表格）」「版本卡片」「特色區①②③④」「功能特色」「適用機型表」「使用情境」這些名稱**都不要再輸出**，對應關係見文末「舊區塊 → 新積木對應」。

## 積木與要產出的文字欄位

1. **Hero（直圖壓文）**（必放）— 兩版型：**A 有品牌**（自有品牌 YU／Londonimg／J/arch.／LaPO 用）／**B 無品牌**（授權經銷、非自有品牌商品用）。標明建議版型。
   - `主標題：`（大字、短，例：All-in-One）
   - `副標題：`（一句賣點，例：一個殼＝防摔＋立架＋指環，解決所有痛點！）
   - 版本 A 另給 `品牌：`（例：YU）

2. **重點 icon 列**（**預設要產**，4 個）— 每個圖示一組：
   - `圖示標題：`（例：360° 環繞支架）
   - `第一行說明：`＋`第二行說明：`（各一短句）
   -（圖示圖檔：同事自備線條 PNG 上傳媒體庫，skill 不用管）

3. **影音**（選用，**有商品影片才放**）：
   - `英文小標：`（例：In Action）、`區塊標題：`（例：影音實測）
   - 每支一組 `影片N標題：`（可英＋中雙語；只有一支就只給一組）
   -（影片本身：提醒同事到 YouTube「分享→嵌入」複製網址自行貼上，skill 不用編）

4. **特色區（多圖）**（**預設要產**；一個賣點＝一塊，可出 2–3 塊，每塊配一排 3 張方圖）：
   - `英文小標：`（例：Stable Stand）
   - `區塊標題：`（中文大標，例：360° 環繞支架 ‧ 持久耐用）
   - `一句說明：`

5. **情境 橫圖壓文**（**預設要產**，滿版大圖疊字、營造使用情境）— 兩版型：**A 文字左上／B 文字左下**。⚠️ A/B 取決於**圖片暗部位置**、skill 沒看過圖 → **不要自己選版型**，輸出這條自選規則給同事：「圖暗部在左上→A、暗部在下方→B」：
   - `英文小標：`（例：Everyday）
   - `情境大標：`（例：解放雙手，生活更自在；也可放產品全名當「優點開場」過場，例：YU 360° 無段支架手機殼）
   - `一句說明：`

6. **棋盤格**（**預設要產**，3–4 列、圖文各半左右交替；逐點講功能或情境）— 兩版型：**A 無英文小標／B 有英文小標**。版型判準：**講功能規格的列→建議 B**（配英文小標，例 MagSafe／Drop Protection）、**講生活情境的列→建議 A**（純中文標題）：
   - 每列一組：`標題：`（例：內建強效 MagSafe 磁吸圈）＋`說明文字：`（一句）
   - 用 B 版時每列另給 `英文小標：`（例：MagSafe／Drop Protection）

7. **多圖橫移（方形滑塊）**（**預設不產**；使用者說有多角度實拍圖／系列品要做圖牆才產。一排可左右滑的正方形圖）：
   - `英文小標：`（例：Gallery）、`標題：`（例：多角度實拍，細節看得更清楚）
   -（圖與連結同事自填；版型 A 有標題／A-2 有標題＋每張可連結／B 只有照片——**建議 B 版時無文字欄位**，輸出一行「【積木：多圖橫移（方形滑塊）】建議版型 B：無文字欄位，直接照教學頁複製語法」即可）

8. **版本比較**（選用，有「一般款 vs 升級款」之類版本差異才放）— ⚠️ **不做 HTML 表格**，改「①合成圖＋②選哪款卡」兩段（可只用其一）：
   - **① 對比合成圖**：`英文小標：`（例：Compare）、`問句標題：`（例：一般款？進化 3.0？）、`一句：`（例：差別在哪呢？）
     -（合成圖本身請美編把規格差異直接畫進圖裡，skill 只產上面文字）
   - **② 選哪款卡**（每版本一張卡）：`英文小標：`（例：Two Versions）、`標題：`（例：一般款 vs 3.0 超進化）；然後**每版本分組**列出：`版本N標籤：`（例：一般款）＋`適合誰→選這款：`（例：要追劇、視訊免手扶 → 選支架款）＋`這個版本的說明：`（⚠️ 教學頁可見欄位就叫「這個版本的說明」；「版本N圖片說明」是圖的 alt、同事自填，別混用）

9. **加價購**（全店共用固定 15 項）— **skill 不產文字**；輸出時只給一行提醒：「此塊全店共用，直接到教學頁『加價購』積木複製整段貼上即可」。

10. **收尾 CTA**（必放；**文字是給 Google 的內容，SEO/GEO 重點**）：
    - `總結大標：`（例：一個殼搞定）＋`副標：`（例：追劇・視訊・防摔・磁吸）
    - `一句描述：`（**品牌＋核心賣點＋適用機型**的完整宣告句——這句承接舊「定義句」的 GEO 任務：主詞清楚、事實密度高、無口語無 emoji、可被 AI 整句引用；**適用機型一定要寫進來**）
    - `按鈕網址：`（該商品完整網址＋`#sl-product-image`；不知道網址就標「請填商品完整網址＋#sl-product-image」，⚠️ 不能只填 #）
    -（按鈕文字模板固定為「回上方選你的機型・立即加入購物車」；商品沒有「選機型」情境時，提醒同事把按鈕裡的字改成合適字句（例：回上方・立即加入購物車），**只改字、不動樣式**）

11. **規格**（**預設要產**）— 每列 `項目：內容`（材質／尺寸／重量／充電…項目數依商品）。

12. **QA**（必放，4–6 則、預設 5）— 每則一組 `問題N：`＋`答案N：`：
    - 問題口語化（用真實問法）；**答案第一句先直接給結論**（GEO 可擷取），再補口語。
    - **適用機型與相容性差異放這裡講**（例：適用哪些機型、mini 為何除外、某機型不支援 MagSafe／僅一般無線充電）；機型不確定時標「請確認」，不要亂編。

13. **SNS 社群圖牆**（**預設不產**，使用者說要放才產；IG 風方形照橫滑牆。教學頁此塊目前標「施工中 🚧」但複製語法已完整可用——產出時順帶跟同事講一聲，免得看到施工中不敢用）：
    - `英文小標：`（例：As Seen On IG）、`區塊標題：`（例：大家都這樣用）
    - IG 帳號固定＝ `@london_apple`（https://www.instagram.com/london_apple/ ——注意不是 londonimg）
    - ⚠️ **只放真實照片**（品牌 IG 或顧客實拍），不可 AI 生成圖、不可配假讚數／假留言。

## 舊區塊 → 新積木對應（左邊的名稱一律不要再輸出）

| 舊名稱（廢除） | 內容改放哪裡 |
|---|---|
| 首屏 Hero 大圖 | **Hero（直圖壓文）**（A 有品牌／B 無品牌） |
| 定義句（給 Google／AI 讀） | **收尾 CTA 的「一句描述」**（同樣寫成完整宣告句） |
| 特色區①②③④ | **特色區（多圖）**（一個賣點一塊）或 **棋盤格**（逐點交替） |
| 對照表（背景圖＋比較表格） | **版本比較 ① 對比合成圖**（差異請美編畫進圖，不做 HTML 表格） |
| 版本卡片（兩張帶標籤卡片） | **版本比較 ② 選哪款卡** |
| 使用情境（多個情境） | **棋盤格**（情境逐列）＋**情境 橫圖壓文**（大場景） |
| 功能特色（背景圖疊三行字） | **情境 橫圖壓文**（滿版大圖疊字） |
| 適用機型（可見表格） | **QA**（問答講機型與相容性）＋ **收尾 CTA 一句描述**帶到機型範圍 |

## 輸出格式（給同事）

- 每個積木用 `【積木：◯◯】` 當小標，底下列出該積木各欄位的文字。**◯◯ 一律用教學頁積木名**：Hero（直圖壓文）／重點 icon 列／影音／特色區（多圖）／情境 橫圖壓文／棋盤格／多圖橫移（方形滑塊）／版本比較／加價購／收尾 CTA／規格／QA／SNS 社群圖牆。
- **有版型之分的積木要標建議版型**（同事才知道複製哪一版語法）：Hero A 有品牌／B 無品牌；棋盤格 A 無英文小標（情境列）／B 有英文小標（功能列）；多圖橫移 A／A-2／B；版本比較 ①／②。**例外：情境 橫圖壓文不標建議版型**，改輸出自選規則（圖暗部左上→A、暗部下方→B）。
- **每個欄位都用教學頁的填空名稱標示**（如 `英文小標：`、`區塊標題：`、`主標題：`、`副標題：`、`情境大標：`、`標題：`、`說明文字：`、`一句說明：`…），讓同事一眼對到教學頁的 ✏️【…】，不要用「｜」混排。
- **不要自加「備註」**：各積木只產該積木欄位，別重複其他積木已講的（相容性差異 → 放 QA）。
- 只輸出「這個商品用得到」的積木；用不到的直接不要出現。
- 不要提 H2/HTML/CSS 等字眼；用「大標」「小標」這種白話。
- 結尾提醒：到教學頁「商品頁語法」找同名積木，把文字填進 ✏️【…】；圖片自己上傳。**也提醒：要可直接貼進後台的 HTML，可再說「也給我可貼的 HTML」（分積木給）。**
