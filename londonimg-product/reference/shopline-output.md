# SHOPLINE 平台限制與輸出格式

> 產出文案／HTML 時務必遵守，否則貼進後台會跑版或被濾掉。（皆為實測結論）

## 後台貼放位置
- **SEO 四欄**：商品 → 編輯 → **網路搜尋最佳化**。標題／簡介／關鍵字／描述性URL 各一欄。
- **商品描述**：商品 → 商品描述 → 右上角 `</>` 切「HTML 原始碼」模式再貼。

## 商品描述欄位會「淨化」HTML（重要・皆實測）
- `<style>`、`<script>` → **整段被移除**（外掛 CSS/JS 全失效）。
- `<details>` / `<summary>` → **連標籤都被濾掉**，折疊失效 → FAQ 要做「卡片式平鋪」（全展開，全文可見也利 SEO）。
- `<svg>` → **被濾掉** → 圖示一律用外部 PNG（上傳 SHOPLINE 媒體庫取得 `https://img.shoplineapp.com/...` 網址）。
- `data:` URI（base64 內嵌圖）→ **被濾掉** → 只能用外部 https 圖片網址。
- CSS `gap` → **被濾掉** → 間距一律改用 `margin`。
- CSS `aspect-ratio` → **被濾掉** → 等比框改用 `padding-bottom:◯%` 比例盒＋內層圖絕對定位 `object-fit:cover`（正方形＝`padding-bottom:100%`、9:16＝`177.78%`）。
- `<iframe>`（YouTube 影片）→ **保留**，可放心用。
- 連結**不可用 `href="#"`**：SHOPLINE 有 `<base href="/">`，相對 `#` 會導去首頁 → 回購買區按鈕一律填**完整網址**＋`#sl-product-image`。

## 單位一律用 px，不要用 rem
- SHOPLINE 主題設了 `html{font-size:62.5%}`，貼進來的 `rem` 會被縮成 62.5%（圖變小、字變小）。
- 響應式改用 `clamp()` / `vw` / `%`（不受根字級影響）；固定尺寸用 px。

## 若要產出「可貼 HTML」（使用者要求時）
- ⚠️ **最高原則：每塊一律照 `reference/block-html-templates.md` 對應積木（對應版型）的模板複製**——模板＝教學頁複製語法的**原樣鏡像**。只把文字類 `✏️【…】` 換成你產的文字；**圖片網址、圖片說明（alt）、YouTube 嵌入網址、貼文／商品連結等佔位照模板原樣保留**（各積木佔位字不同，別改寫、別統一、別加序號；alt 由同事換圖時自填）。**兩個例外直接代入**：① SNS 的 IG 帳號／網址＝`@london_apple`／`https://www.instagram.com/london_apple/`；② 收尾 CTA 兩顆按鈕的網址——已知商品網址就代入「完整網址＋#sl-product-image」（兩處同值），未知才留佔位並提醒補填。**不可自創版面、不可改任何 style、不可加 font-family**（唯一可改的非 ✏️ 文字：收尾 CTA 按鈕字樣——商品無「選機型」情境時可改成合適字句）。下面其餘規則都建立在「用那份模板」的前提上。
- ⚠️ **一律分段給，不要給一整包**（與教學頁一致）：**每個積木各自一個獨立程式碼框**，讓同事像教學頁那樣「需要哪塊就複製哪塊」。**絕不可**把全部積木塞進同一個 `<div>` 一次給。
- **每個積木框的開頭標出積木名稱＋版型**（用教學頁同名），例如：
  - `【可貼 HTML — Hero（直圖壓文）‧ 版本 A 有品牌】` ＋ 其下一個 ```html 框
  - `【可貼 HTML — 棋盤格 ‧ 版本 B 有英文小標】` ＋ 其下一個 ```html 框
- **每塊都要能單獨貼、單獨成立**：每塊是一個自成一體的 `<div style="…">…</div>`，同事單獨複製貼進商品描述就能正常顯示，不依賴其他塊。
- **寬度照模板、不要自行加減 max-width（重要）**：v3 設計＝內容窄欄 `max-width:1080px` 置中＋滿版積木（情境 橫圖壓文／棋盤格／多圖橫移／加價購／SNS 橫滑列）用 `width:100vw;left:50%;margin-left:-50vw` 破出描述容器（實測 SHOPLINE 描述容器約 1217px、比視窗窄，不破出就不滿版）。**這些都已寫死在模板裡，照抄即可**；自己加/刪 max-width 或滿版設定都會跑版。
- **按積木順序由上到下給**：Hero（直圖壓文）→ 重點 icon 列 → 影音 → 特色區（多圖）→ 情境 橫圖壓文 → 棋盤格 → 多圖橫移（方形滑塊）→ 版本比較 → 加價購（提醒照教學頁複製，不產 HTML）→ 收尾 CTA → 規格 → QA → SNS 社群圖牆。同事依序往下貼即是正確順序；要跳過某塊直接不複製即可。
- **全 inline style**（每個元素 `style="…"`），不要外部 CSS／不要 `<style>`。
- 響應式靠 `clamp()`＋`flex-wrap`＋`padding-bottom` 比例盒內嵌，**不依賴 @media**。
- 同事流程：先把圖上傳 SHOPLINE 取得 `https://img.shoplineapp.com/...`，再搜尋 ✏️ 把每個佔位換成網址；同一積木多張圖照「上到下／左到右」順序對應即可。

## 輸出呈現（給同事複製）
- 每個欄位／積木用**獨立程式碼框**，方便一鍵複製、按需挑用。
- SEO 四欄各自一框。
- **純文字版**：每塊用 `【積木：◯◯】` 當小標，**其下各欄位分別給獨立程式碼框**（同事逐格複製填 ✏️）。
- **可貼 HTML 版**：每塊用 `【可貼 HTML — ◯◯ ‧ 版型】` 當標題，**一積木一個 ```html 框**（整塊複製）。
- 兩種版本都**不要把全部積木塞成一整包**。

## 文案產出後提醒使用者
- 改了 slug → 去設 **301 轉址**。
- 圖片網址要先上傳 SHOPLINE 再複製；貼完 HTML 後，**搜尋 ✏️ 把每個圖片／影片／連結佔位換成真實網址、圖片說明（alt）順手補上**（還留著 ✏️ 代表那格還沒換）。
- 描述欄貼 HTML 前先切「HTML 原始碼」模式。
