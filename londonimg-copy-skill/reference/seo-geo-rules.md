# SEO / GEO 規則（各欄位寫法）

> 萃取自 seo-geo 方法論，套用到倫敦印象 SHOPLINE 商品頁。

## 1. 標題（meta title）

- 結構：**品牌 ＋ 機型/品類 ＋ 核心功能**（＋型號可放末尾）。
- 長度：約 **28–32 個全形字**（Google 行動版約顯示前 30 字）。
- 必含主關鍵字（使用者搜尋會打的詞，如「iPhone 手機殼」「支架手機殼」）。
- **禁止**：促銷字（任選兩件 88 折…）、花式 Unicode 字體、emoji、全形驚嘆號堆疊。
- 範例：`YU iPhone 360°無段支架手機殼｜MagSafe 防摔殼 12-16 系列【B67】`

## 2. 簡介（meta description）

- 長度：**約 150 字**（Google 約顯示前 78 字 → 重點前置）。
- 內容順序：品牌＋核心功能 → 次要功能 → 適用機型 → 材質/特點 →（可選）促銷/免運。
- 一句話講清楚「這是什麼、給誰、為何選它」。自然帶入關鍵字，勿堆砌。
- 範例：`YU 360°無段支架 iPhone 手機殼，內建金屬指環支架可任意角度追劇、視訊免手扶；支援 MagSafe 磁吸，四邊與鏡頭框墊高防摔。適用 iPhone 12～16 全系列（mini 除外）。TPU+PC 複合材質、PU 觸感防汙好握。指定款任選兩件 88 折，滿 $599 超取免運。`

## 3. 關鍵字（keywords）

- **半形逗號分隔**，8–12 個。
- 涵蓋：品類詞、機型詞（逐代 iPhone16/15/14…）、功能詞、品牌詞、長尾詞、型號。
- 範例：`iPhone手機殼,360支架手機殼,MagSafe手機殼,指環支架手機殼,防摔殼,iPhone16手機殼,iPhone15手機殼,YU手機殼,追劇支架殼,B67`

## 4. 描述性URL（slug）

- **只用小寫英文與數字、連字號 `-`**。禁止中文、空格、`°`、底線。
- 結構：`品牌-機型語意-品類`。
- 範例：`yu-iphone-360-ring-stand-case`
- ⚠️ **改 slug 會變動網址** → 一定要提醒使用者去後台設 **301 轉址**（舊網址→新網址），否則既有連結與 Google 排名會斷。

## 5. 定義句（GEO，給 AI/精選摘要引用）

- **一句完整的宣告句**：`〔品牌+商品名〕是一款〔特色1〕、〔特色2〕、適用〔機型〕的〔品類〕，〔一句賣點〕。`
- 要能被 AI 整句引用 → 主詞清楚、資訊密度高、不要分散成多句。
- 範例：`YU 360°無段支架手機殼是一款內建金屬指環支架、支援 MagSafe 磁吸、適用 iPhone 12–16 系列（mini 除外）的防摔手機殼，可任意角度直立／橫放，一殼搞定追劇、視訊與防摔。`

## 6. 段落標題結構（商品描述分區）

> 對同事說「段落大標」即可；產出可貼 HTML 時，這些標題用 `<h2>`（利 SEO），但**不要對同事提 H2/標籤名**。

建議固定四區，利讀者掃讀也利 SEO：
1. **商品特色**（3–4 個 bullet，每個＝功能＋好處）
2. **適用機型**（做成**可見表格** → 同時餵 GEO）
3. **規格**（材質、尺寸、充電方式…）
4. **湊單提醒**（AOV：滿額免運／任選折扣，引導加購）

## 7. FAQ（5 則，可掛 FAQ Schema）

- 挑使用者真的會問的：相容機型、功能怎麼用、耐用度、相容配件、版本差異。
- 問題口語化、答案具體；**頁面要可見**才可掛 FAQ schema。

## 自我檢查清單

- [ ] 標題 ≤ ~32 全形字、含主關鍵字、無促銷字/花式字體
- [ ] 簡介 ~150 字、重點前置
- [ ] 關鍵字半形逗號、含逐代機型
- [ ] slug 全英數小寫、無特殊字元 ＋ 已提醒 301
- [ ] 定義句是「一句」完整宣告句
- [ ] 適用機型用表格呈現
