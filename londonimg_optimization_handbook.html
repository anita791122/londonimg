01

商品篩選器（Faceted Filter）

轉換率 UX
問題 你有上千款商品，但顧客最常見的任務是「我的 iPhone 16 Pro，要 MagSafe、藍色、防摔」。目前他們只能一層層點分類，無法交叉篩選。這是配件電商最大的轉換漏點——找不到就離開。

SHOPLINE 內建做法：用「商品標籤 + 自訂篩選」

SHOPLINE 後台的商品篩選是靠 商品標籤（Tags） 與 商品分類 驅動的。要做出好用的篩選器，關鍵不是寫程式，而是把每一個商品都打上一致、結構化的標籤。標籤亂，篩選器就廢。

操作位置 後台 商店設定 → 商品分類 / 篩選設定，以及每個商品的 商品管理 → 編輯商品 → 標籤欄位

建立一套固定的標籤命名規則（最重要）

請對全站商品套用以下五個維度的標籤。維度前綴讓篩選器能正確分組，請務必統一用詞、不要同義詞混用（例如「防摔」就不要有人寫「抗摔」）。

篩選維度標籤命名範例（請固定用詞）
適用機型機型-iPhone17Pro、機型-iPhone16、機型-AppleWatch46mm
是否 MagSafeMagSafe-支援、MagSafe-不支援
防護等級防護-軍規防摔、防護-一般、防護-超薄
顏色色-黑、色-透明、色-藍、色-奶茶
價格帶用 SHOPLINE 內建價格篩選即可，不需另打標籤
為什麼這樣設計 前綴（機型- / 色- / 防護-）會讓你在開啟「依標籤篩選」時，能把同類標籤歸到同一個篩選群組，顧客看到的就是「機型 / 顏色 / 防護」分區的勾選框，而不是一長串混在一起的標籤。

開啟分類頁的篩選側欄

  1. 進入 網店設計 → 佈景主題 → 自訂。
  2. 找到「商品分類頁」版型，開啟 「顯示篩選 / 排序工具列」 選項（SHOPLINE 多數佈景主題內建此功能）。
  3. 在篩選設定中，把上面建立的標籤維度加入可篩選欄位。
  4. 確認排序選項包含：最新、熱銷、價格低到高、價格高到低。
若佈景主題沒有內建篩選 部分舊版 SHOPLINE 佈景沒有完整的標籤篩選。若是如此，最務實的替代方案是建立「組合分類」——例如直接做一個分類叫「iPhone 16 Pro ｜ MagSafe 防摔」，把符合的商品都歸進去，並放進選單。這雖然是手動的，但對最熱門的機型組合（17 Pro、16 Pro 系列）效益最高。

驗收標準

  • 任選一個熱門機型分類頁，左側 / 上方出現「機型、顏色、MagSafe、防護」篩選選項。
  • 勾選「MagSafe-支援 ＋ 色-透明」後，列表確實只剩符合的商品。
  • 手機版篩選按鈕可正常開合，不擋住商品。
  • 每一個在售商品都至少有「機型、顏色、MagSafe」三個維度的標籤。
02

商品標題清理 ＋ 社會證明前置

信任 SEO
問題 目前商品標題塞滿促銷詞：「熱銷NO.1」「店長推薦👍」「瞬！超爽…」，甚至出現「新機必備 新機必備」重複。短期吸睛，長期降低信任、也稀釋 SEO 關鍵字（Google 看到的是一堆形容詞而非產品本質）。

商品標題公式

把促銷詞從標題拿掉，改用「品牌 ＋ 產品線 ＋ 核心賣點 ＋ 型號」的固定結構。促銷感改用「商品標籤 / 圖片角標」呈現，不佔標題。

現況 ✕ 新機必備 新機必備 無敵防摔 極地冰岩殼 iPhone 手機保護殼+玻璃貼Londonimg【K50s】
建議 ✓ Londonimg 極地冰岩殼 軍規防摔手機殼（附玻璃貼）iPhone 17/16 全系列【K50s】
為什麼 「軍規防摔」「附玻璃貼」「適用機型」都是顧客真的會搜尋的字，放進標題能吃到搜尋流量；而「熱銷NO.1」這種字沒人會搜。促銷急迫感改用下面的角標處理，效果更好且不傷品牌。
操作位置 標題：商品管理 → 編輯商品 → 商品名稱　｜　角標：商品管理 → 標籤 或佈景主題的「商品角標 / Badge」設定

促銷感改用角標（Badge）呈現

把「熱銷」「店長推薦」「新品」做成商品圖左上角的小標籤，而不是寫在文字標題裡。SHOPLINE 佈景主題多半支援用特定標籤自動顯示角標，例如把商品打上 角標-熱銷、角標-新品。

社會證明前置

配件客單低、決策快，「別人也買 / 別人說好」能直接推動衝動購買。目前首頁與商品卡幾乎看不到評價或銷量，建議補上。

  1. 開啟商品評價功能。前往 App Store（SHOPLINE 應用程式商店）→ 安裝商品評論 App，讓買家可留星等與照片。
  2. 把評價星等顯示在商品卡。在佈景主題自訂中開啟「商品列表顯示評分」。
  3. 首頁加一區「真實開箱 / 評價牆」。你已有部落格「倫印3C開箱趣」，把其中幾篇拉到首頁區塊，或用評論 App 的精選評價輪播。
  4. 熱賣區改用真實數字。把「熱銷NO.1」這種說法，盡量換成可信的具體呈現，如「累積銷售 ⭐4.8（1,200+ 則評價）」。

驗收標準

  • 前 20 個熱賣商品的標題已移除重複字與純促銷詞，符合新公式。
  • 商品列表頁的卡片上可看到星等評分。
  • 首頁有一個明顯的評價 / 開箱區塊。
  • 促銷感以角標呈現，標題乾淨。
03

Product / FAQ 結構化資料（Schema）

SEO GEO
問題 你的網站目前沒有 Product / FAQ 的結構化資料（Schema）。加上之後，Google 搜尋結果能直接顯示「價格、星等、庫存」的 rich snippet，而 ChatGPT / Perplexity / Google AI 摘要也更容易引用你的店——這同時吃 SEO 與 GEO。
什麼是 Schema（白話版） 它是一段藏在網頁裡、給機器看的「標準格式說明卡」。人類看不到，但 Google 與 AI 看得到。它明確告訴機器：這是一個「商品」，名稱叫 X、價格 490、有 4.8 顆星。機器有了這張卡，就更願意把你的資訊放進搜尋摘要或 AI 回答裡。

3-A ｜ Product Schema（商品結構化資料）

貼在哪裡 後台 網店設計 → 偏好設定 / 自訂程式碼 → 商品頁（Product page）的 <head> 或頁尾。SHOPLINE 商品頁支援帶入商品變數，下面用「{{ }}」標出需要動態帶入的欄位；若你的佈景不支援該變數，就改成手動填寫代表性數值。
product-schema.html　— 貼在商品頁
<script type="application/ld+json">
{
  "@context": "https://schema.org/",
  "@type": "Product",
  "name": "{{商品名稱}}",
  "image": "{{商品主圖網址}}",
  "description": "{{商品描述前 120 字}}",
  "sku": "{{商品編號，如 K50s}}",
  "brand": {
    "@type": "Brand",
    "name": "{{品牌，如 Londonimg / 犀牛盾}}"
  },
  "offers": {
    "@type": "Offer",
    "url": "{{此商品頁網址}}",
    "priceCurrency": "TWD",
    "price": "{{售價數字，如 490}}",
    "availability": "https://schema.org/InStock",
    "itemCondition": "https://schema.org/NewCondition"
  },
  /* 有評價才放下面這段，沒有就整段刪掉，否則 Google 會判定造假 */
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "{{平均星等，如 4.8}}",
    "reviewCount": "{{評價則數，如 126}}"
  }
}
</script>
重要 aggregateRating 那段只有在真的有評價時才放。沒有評價卻填數字，Google 會判定為灌水，反而懲罰你的排名。所以建議先做完第 2 項（開啟評價），再回來補這一段。

3-B ｜ FAQ Schema（常見問題結構化資料）

你已有「購物常見問題」頁，這是 AI 摘要最愛抓的格式。把它包成 FAQPage Schema，被 ChatGPT / Google AI 引用的機率大幅提升。

貼在哪裡 後台 頁面管理 → 編輯「購物常見問題」頁 → 切換到 HTML / 原始碼模式 → 貼在內文最下方
faq-schema.html　— 貼在常見問題頁
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "訂單滿多少免運費？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "全館消費滿 NT$599 即享超商取貨免運。"
      }
    },
    {
      "@type": "Question",
      "name": "手機殼可以退換貨嗎？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "商品到貨 7 天內、保持完整包裝可申請退換貨，詳見退換貨政策。"
      }
    },
    {
      "@type": "Question",
      "name": "出貨需要幾天？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "訂單成立後 1～2 個工作天內出貨。"
      }
    }
  ]
}
</script>

↑ 把問與答換成你真實的政策內容即可，問題可以多加幾題（運費、保固、發票、MagSafe 相容性等）。

3-C ｜ 驗證有沒有生效

  1. 貼完並儲存後，用 Google 官方工具測試：Rich Results Test（搜尋「Google Rich Results Test」進入）。
  2. 貼上你的商品頁 / FAQ 頁網址，看它是否偵測到 Product / FAQPage 且無錯誤。
  3. 幾天後在 Google Search Console 的「強化」報表確認沒有錯誤。

同場加映 ｜ 給 AI 引用的內容（GEO）

除了 Schema，AI 偏好引用「結構清楚、含具體數據」的內容。建議在「品牌介紹」或部落格新增一段可被擷取的事實段落，例如：

  • 明列官方授權品牌清單（犀牛盾、UAG、Spigen…）— AI 回答「哪裡買得到犀牛盾」時會比對。
  • 自有品牌的可量化賣點：例如 D12「厚度 0.8mm」、K50s「軍規 X 米防摔測試」、行動電源「10000mAh / 22W」— AI 回答「最薄的 MagSafe 殼」時會抓這些硬數據。
  • 寫一篇「2026 iPhone 配件選購指南」長文，完整回答「該選防摔還是輕薄」「MagSafe 值不值得」等問題並內鏈到商品。
04

湊單與組合定價策略

客單價
問題 目前唯一湊單誘因是「滿 599 免運」，但手機殼單價約 490——買一個就快免運了，等於沒有動機加購第二件。客單價被卡在「一件」。

三個可立刻設定的機制

① 階梯式免運／折扣，把門檻拉高一階

把「滿 599 免運」維持為基本盤，再疊一層更高門檻的折扣誘因，引導顧客多放一件進購物車。

門檻誘因用意
滿 599超取免運（維持現況）基本盤
滿 990折 100 ／ 送鏡頭貼把「買一個殼」推成「殼＋玻璃貼＋鏡頭貼」
滿 1,500免運（含宅配）＋ 贈掛繩拉動高客單（含 Apple Watch 錶帶／行動電源）
操作位置 促銷管理 → 新增促銷活動 → 滿額折扣 / 滿額贈品 / 滿額免運

② 商品頁加購（Add-on）綁定

在每個手機殼商品頁，設定「常一起買」的加購品：玻璃貼、鏡頭貼、掛繩。顧客在結帳前一步看到，加購率很高。

操作位置 商品管理 → 編輯商品 → 加購商品 / 推薦商品設定

③ 把組合包前置到首頁

你已經有「組合包專區」（手機殼組合、錶框錶帶組合、充電組合），但首頁沒主動推。組合包的「比分開買省 X 元」訊息是最強的客單推手，應該放上首頁主視覺或熱賣區第一排。

定價的隱性問題 順帶提醒：目前幾乎所有商品都掛著劃線折扣（如 NT$490 NT$590）且長期出清 3 折。當「全部都在打折」，折扣就失去說服力，也壓低自有品牌的價值感。建議把商品分成兩條線——常態品（不掛折扣、講品質與保固） 與 出清品（集中在出清專區打折），避免主力商品被自己的折扣標籤拉低形象。

驗收標準

  • 結帳購物車會顯示「再買 OOO 元可享 ___」的進度提示（多數佐景主題的滿額活動會自動顯示）。
  • 手機殼商品頁底部出現玻璃貼 / 鏡頭貼加購選項。
  • 首頁可看到組合包區塊與「省下多少」的訊息。
  • 觀察 4 週後，平均每筆訂單件數是否從 ~1 件上升。
05

導覽層級簡化（手機版）

UX 降跳出
問題 主選單下「品牌嚴選」一口氣列出近 50 個品牌、「手機殼」列出 30+ 機型，下拉變成一面牆。手機版尤其難滑，顧客還沒找到就放棄。

兩段式收合：先選世代，再選型號

把「手機殼」「鏡頭貼」這類超長機型清單，改成先選世代（17 / 16 / 15…），點開才展開該世代的型號。你的後台其實已經有「iPhone 17 系列 / 16 系列」的群組分類，只是首頁主選單沒用上——把選單改成連到「系列」層即可。

現況 ✕ 手機殼 ▸ 直接列出 iPhone 17 Pro / 17 Pro Max / 17 / Air / 16 / 16 Plus …（30+ 項一次攤開）
建議 ✓ 手機殼 ▸ iPhone 17 系列 / 16 系列 / 15 系列 …（點世代才展開型號）
操作位置 網店設計 → 導覽選單 / 選單管理，把子項目改成連到「系列」分類頁，型號收為第三層。

品牌頁改用 Logo 牆 ＋ 搜尋

近 50 個品牌用純文字清單很難掃讀。把「品牌嚴選」主項目直接連到一個品牌總覽頁（你已有 /categories/brand），頁面用品牌 Logo 的格狀排列＋可搜尋，比下拉一長串好用得多。選單裡只保留最熱門的 6～8 個品牌即可。

手機版優先順序

手機版的選單，把最高頻入口放最前面：

  1. 依機型找殼（最高頻）
  2. 熱賣 / 新品
  3. 出清特賣
  4. 品牌總覽
  5. Apple Watch / 充電周邊
  6. 其他（關於我們、購物須知收進次選單）

順手修掉的小瑕疵

頁面渲染瑕疵 抓取首頁時發現有未正常顯示的樣板變數，以及商品名稱出現 F&amp;amp;W 這類重複轉義的亂碼（應為 F&W）。請在商品名稱、活動文案中搜尋 &amp; 並修正，這同時影響觀感與 SEO。

驗收標準

  • 手機版打開「手機殼」選單，第一層只看到世代（17/16/15…），不再是 30 項型號。
  • 「品牌嚴選」連到 Logo 牆總覽頁，可搜尋。
  • 商品名稱中不再出現 &amp; 亂碼。
  • 手機版單手滑動就能在 2～3 次點擊內找到目標機型的殼。
